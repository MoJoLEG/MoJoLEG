Classification heuristics (apply in order; tie-break by narrative centrality):
• Dialogue speakers (name labels like “민수:”) ⇒ Main.

• Frequency/salience in the scene: appears in multiple sentences, performs pivotal actions, holds key props linked to plot ⇒ Main.
• Brief presence, background actions, or mentioned without direct action ⇒ Sub.
• When only one character exists: place in Main_Characters.
• Hand-prop holders referenced in props.hand_props (see below) must be present in characters; default them to Main if they also speak or drive the action; otherwise Sub.

– Items under “미상” are physically present and actively used/handled in the scene but cannot be confidently attributed to a specific character. If there are no unassigned items, include “미상”: [].

– Inclusion limits: only items physically present and used/held in the current scene (exclude plans/ads/future mentions).
– Binding rule (to decide which character key gets an item): If a sentence contains both an item and one or more character names and includes a hold/wear/pass verb, assign that item to all such characters; without such verbs but strong co‑occurrence, assign to the first mentioned character in that sentence.

[SCENE SPLITTING — STRICT]
A new scene starts only at a line that matches one of these exact header forms (case-insensitive for SCENE), with no extra text on the same line:
• ^S\d+[A-Z]?$  → e.g., S12, S12A
• ^SCENE\s+\d+[A-Z]?$  → e.g., SCENE 5, SCENE 21A
• ^#\s*\d+[A-Z]?$  → e.g., #7, #101B
Disallowed as scene headers (do not split on these):
• Numeric dot forms like 12. INT. …
• # 씬 7, Scene #7, or any line with additional tokens beyond the allowed forms
• Any heuristic that “looks like a header” but doesn’t match one of the three patterns above
If none of the allowed headers appear, treat the entire input as a single scene with scene_number=1.
